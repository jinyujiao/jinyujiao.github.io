记录Github+Hexo搭建博客的过程

发表于 2019-01-06 | 分类于 技术

准备工作

  1. 下载安装Node.js：Node.js官网，一路next就可以了。
    安装好以后打开Cmd，输入
    node -v以及npm -v
    能正确显示版本号即为安装成功。
    国外的源速度实在太慢，所以我们切换全局配置到淘宝源：
    npm config set registry https://registry.npm.taobao.org
  2. 安装Git：Git官网
    安装成功的象征就是在电脑上任何位置鼠标右键能够出现如下两个选择：
  3. 注册Github,然后创建新项目，项目名称必须是用户名.github.io这种格式。

    这边创建好就可以先放着了，待会才会用到它。

安装HEXO

在自己认为合适的地方创建一个文件夹，至于名字嘛，自己喜欢就行（如E:\bolg）。进入刚刚创建的文件夹，右键鼠标，选择git bash，输入：
npm install hexo-cli -g

接下来的所有命令都是在刚刚我们创建的文件夹内使用(比如我的E:\bolg)。
安装完成后可以输入 hexo -v 来检查是否安装成功。

接着输入hexo init ，初始化该文件夹，看到后面的“Start blogging with Hexo！”的话表示初始化完成，那我们的工作已经成功了一大半啦！
最后安装输入npm install ，安装所需的组件。
接下来就是激动人心的时刻啦！
输入：hexo g（hexo generate的简写）生成。
再输入hexo s （hexo server的简写）。

假如页面一直无法跳转，那么可能端口被占用了。此时我们ctrl+c停止服务器，接着输入hexo server -p 端口号来改变端口号
打开后差不多是下图的样子就说明安装成功了：

将Hexo关联到Github Page

  1. 设置github的user name和email
    在我们创建的blog文件夹右键，选择git bash，输入(名字和邮箱为刚刚注册GitHub时用的)：

  2. 生成ssh密钥
    输入cd ~/.ssh
    再输入ls,列出该文件夹下面的内容，

    该图说明文件夹存在，而且下面已经有好几个文件了，如果该文件夹不存在，那么直接进行下一步。
    再输入
    ssh-keygen -t rsa -C "your email"
    连续三个回车，生成密钥，最后得到了两个文件：id_rsa和id_rsa.pub（默认存储路径是：C:\Users\当前用户名\.ssh）。
    找到上述的文件夹，打开id_rsa.pub文件，复制里面的内容到github->settings->SSH and GPG key->new ssh key
    具体如下图：

    测试ssh设置是否成功：
    ssh -T git@github.com
    第一次连接或测试的话需要输入个yes，然后最下面显示Hi 你的用户名！ 的话就代表成功了。
    假如ssh-key配置失败，只要按以下步骤就可以解决：

    首先，清除所有的key-pair
    ssh-add -D
    rm -r ~/.ssh
    删除你在github中的public-key
    重新生成ssh密钥对
    ssh-keygen -t rsa -C "xxx@xxx.com"
    接下来正常操作
    在github上添加公钥public-key:
    1、首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
    2、在github上添加公钥时，直接复制即可
    3、保存
    测试：
    在终端 ssh -T git@github.com
